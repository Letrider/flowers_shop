# Flowers Shop Server

Backend API для магазина цветов.

## Установка

```bash
npm install
```

## Настройка

Скопируйте `.env.example` в `.env` и настройте переменные окружения:

```bash
cp .env.example .env
```

### Переменные окружения

- `PORT` - порт сервера (по умолчанию: 4000)
- `JWT_SECRET` - секретный ключ для JWT токенов (обязательно измените в продакшене!)
- `ADMIN_EMAIL` - email администратора по умолчанию
- `ADMIN_PASSWORD` - пароль администратора по умолчанию
- `UPLOAD_DIR` - директория для загрузки файлов (по умолчанию: uploads)

## Запуск

### Режим разработки

```bash
npm run dev
```

### Продакшн

```bash
npm start
```

## API Endpoints

### Авторизация

- `POST /api/auth/login` - вход в админ-панель

### Товары

- `GET /api/products` - список всех товаров (с пагинацией)
- `GET /api/products/:id` - получить товар по ID
- `GET /api/products/by-slug/:slug` - получить товар по slug
- `POST /api/products` - создать товар (требует авторизацию)
- `PUT /api/products/:id` - обновить товар (требует авторизацию)
- `DELETE /api/products/:id` - удалить товар (требует авторизацию)

### Домашняя карусель

- `GET /api/home-carousel` - список слайдов
- `POST /api/home-carousel` - добавить слайд (требует авторизацию)
- `PUT /api/home-carousel/:index` - обновить слайд (требует авторизацию) 
- `DELETE /api/home-carousel/:index` - удалить слайд (требует авторизацию)

### Загрузка файлов

- `POST /api/upload` - загрузить изображение

## Структура данных

Данные хранятся в `data.json` файле. При первом запуске создается администратор по умолчанию с учетными данными из `.env`.
